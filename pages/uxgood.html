Want some revision ideas?

My proposed fix would be to add user input for the student's major, minor, and concentration so the database can be filtered. This filter will be unique to major, minor, and concentration because the classes that will be listed are the classes specifically for the department. This will help users register quicker, easier, and allow them to get back to focusing on what is needed: their present classees.

Users can enter their major and/or concentration, and the subjects table will drop down to the classes that are needed to be taken by that major. Students would no longer have to either scroll through the long drop down, or would they have to enter in the subject.






This design improves because this is the first step that allows the database of the subjects and classes to be filtered. This will allow the user to register for UNA classes faster without having to go to anther page to find out the information for the classes that they want/need to take.
